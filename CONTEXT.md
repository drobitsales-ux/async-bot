## ЦЕЛЬ
Алготрейдинг крипто-фьючерсов → прохождение отбора проп-компании.
Приоритет №1: защита капитала (daily DD), №2: стабильный PF>1.5.
Главный кандидат на проп: стратегия **SA** (BTC VWAP Mean Reversion).

## АРХИТЕКТУРА
- `async_bot.py` — основной бот, BingX, все стратегии, SQLite trades.db (/data/), Telegram-отчёты.
- `bybit_worker.py` — исполнитель на Bybit (боевой проп-счёт), получает сигналы по HTTP от основного бота. **SA-ONLY** (ALLOWED_STRATEGIES=('SA',), v40).
- Хостинг Render.com. AI-оракул Groq/Llama-8b (подтверждение сигналов, MUST APPROVE-формулировки).
- ПРАВИЛО СИНХРОНИЗАЦИИ: логика управления позициями, внедрённая в async_bot.py, обязана зеркалиться в bybit_worker.py (исторический источник багов).

## СТАТУС СТРАТЕГИЙ (2026-07-13)
- **SA** — live BingX + Bybit. 40 live сд: WR 55%, PF 2.15. Фильтр объёма 1.3–2.0x (жёсткий, v37).
- **SMC** — live только BingX. 24 сд, PF 1.70. Фильтр RSI 55-65 + alt<45 (EARLY, n<15).
- **PB** — shadow. Базово убыточна (PF 0.5–0.8 на 191 сд); edge только ADX≥60 (n=14 < порога 15). НЕ готова к live.

## КЛЮЧЕВЫЕ РЕШЕНИЯ (не пересматривать без новых данных)
1. Победа = net_pnl > 0 (после комиссий), НЕ tp50_hit и НЕ pnl_pct-порог. realized_pnl_usdt ≠ итоговый PnL (это только частичные TP50).
2. Qty: округление к ближайшему шагу лота (ccxt.ROUND, у ccxt дефолт TRUNCATE!). Проп-guard: факт. риск > 1.30× целевого → пропуск сделки.
3. SA Smart Timeout: 75 мин, not tp50_hit, pnl≤0, MFE<0.35% → market close. Пороги ВРЕМЕННЫЕ, калибровка по mfe_time_min при n≥30. Зеркало в воркере.
4. Мораторий на асимметричные Long/Short фильтры SA до n≥30 на направление (p=0.25 на n=40 — не значимо).
5. SMC 14-17h НЕ банить (0/7, P(случайность)=13%).
6. check_daily_reset() в главном цикле воркера (не зависит от сигналов); day_start_bal обновляется при смене дня.

## НАБЛЮДЕНИЯ ИЗ ДАННЫХ (кандидаты, ЖДУТ n≥30)
- VWAP-дист 1.8–2.2 ATR: WR 92%, PF 30.5 (n=12) vs 2.2+: WR 40%, PF 1.47 (n=25). Fisher p=0.003. Кандидат: потолок дистанции ~2.2–2.3 ATR. 2.2+ пока PF>1 — не резать до подтверждения.
- Час 00-06 UTC (Азия): WR 81%, PF 13.1 (n=16) vs остальные 37.5%. Fisher p=0.007. Кандидат: сессионный фильтр/веса.
- MFE таймаутов = 0.24% (vs 1.43% у TP) → «висяки» мертвы с рождения = лечится фильтром ВХОДА, порог smart-timeout 0.35% адекватен.
- HTF-наклон: данных 3 сд (копится с v38) — не трогать.

## ПРИНЦИПЫ АНАЛИЗА
- n≥30 до действия (n≥15 для кандидатов). Fisher exact + Wilson CI. Пересекающиеся CI = фильтра нет.
- Shadow-валидация перед live. Честная оценка важнее оптимизма.
- Сначала вывести метрику в отчёт (данные часто уже в БД!), потом решать.
- Anti-overfit: у фильтра должно быть рыночное обоснование, не только цифры.
