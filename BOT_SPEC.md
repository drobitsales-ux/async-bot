# Торговый бот: описание работы, стратегии, фильтры
> Версия кода: **async_bot v45 / bybit_worker v45** | Документ от 2026-07-23
> Назначение: справочник по логике и параметрам. Обновлять при каждом изменении фильтров.

---

## 1. Архитектура

| Компонент | Роль | Биржа |
|---|---|---|
| `async_bot.py` | Генератор сигналов, все стратегии, БД сделок, отчёты | BingX |
| `bybit_worker.py` | Исполнитель сигналов, **только SA** | Bybit (боевой проп-счёт) |

**Поток:** `async_bot` сканирует рынок → сигнал проходит фильтры → AI-оракул (advisory) → открытие на BingX → HTTP-публикация сигнала воркеру → воркер применяет свой risk-сайзинг и открывает на Bybit.

**Ключевое правило проекта:** логика управления позицией, добавленная в `async_bot.py`, обязана быть зеркально внедрена в `bybit_worker.py`. Расхождения между ботами — исторически главный источник багов (см. §7).

**Изоляция воркера (v40):** `ALLOWED_STRATEGIES = ('SA',)` — на боевой счёт попадает только SA как единственная стратегия с доказанным edge. SMC/RSI работают на BingX для форвард-статистики.

## 2. Стратегия SA — Single-Asset VWAP Mean Reversion (главный кандидат на проп)

**Идея:** BTC на 15m растягивается от внутридневного VWAP на экстремуме RSI → вход против растяжения → цель = возврат к VWAP.

**Инструмент:** только BTC/USDT:USDT. **Таймфрейм:** 15m.

### Фильтры входа (порядок применения в коде)

| № | Фильтр | Значение | Reason | Обоснование |
|---|---|---|---|---|
| 1 | Объём | `1.3 ≤ vol_ratio ≤ 2.0` | `vol_climax` | **Доказан статистически.** 1.3–2x: WR 72–77%, PF 2.25–2.94 (n=44) против 2–4x: WR 10%, PF 0.02 (n=10). Fisher exact **p=0.0002**. Механика: умеренный объём = истощение движения (идеал для MR); всплеск ≥2x = инициация нового импульса с реальным участием, контртренд против него не работает. |
| 2 | Потолок дистанции | `\|dist_atr\| ≤ 2.2` | `vwap_far` | **Доказан статистически.** Зона 1.5–2.2 ATR: WR 74–80%, PF 14–24 (n=23) против 2.2+: WR 34%, PF 1.17 (n=32). Fisher **p=0.007**. Механика: дальше ~2.2 ATR это уже импульсная нога, цена строит новую зону стоимости, а не возвращается к средней. |
| 3 | Минимум дистанции + RSI | `\|dist_atr\| ≥ 1.5` И `RSI ≤ 30` (Long) / `RSI ≥ 70` (Short) | `no_setup` | Базовое определение сетапа: растяжение + подтверждение экстремумом осциллятора. |
| 4 | Минимальный RR | `RR ≥ 0.7` | `low_rr` | Введён в v44: RR < 1 при WR 51% математически убыточен. **⚠️ Порог требует пересмотра — см. §6.** |

### Расчёт уровней

```
sl_pct = clip(ATR/price × 2.0, MIN_SL_PCT=1.0%, MAX_SL_PCT=2.5%)
SL     = price ∓ price × sl_pct
TP     = VWAP (плавающая цель, не фиксированный %)
RR     = |TP − price| / |price − SL|
```

**Следствие, которое надо понимать:** TP плавает вместе с волатильностью, а SL зажат снизу в 1.0%. Во флэте (низкий ATR) TP сжимается, SL — нет, и RR разваливается. Это корень проблемы из §6.

### Логика выхода (порядок приоритета)

| Механизм | Условие | Действие |
|---|---|---|
| **TP50** | цена прошла ≥ 50% пути к TP (мин. 0.2%) | фиксация 50% объёма + SL → БУ (+0.15% на комиссию) |
| **TP50 при неделимом лоте** (v43) | `qty = 0.001` (минимальный лот BTC) | partial невозможен физически → весь объём остаётся, **SL → БУ** |
| **Фронтран БУ** | цена прошла 80–100% пути к VWAP | превентивный перевод SL в БУ |
| **Smart Timeout** (v38/v42) | `>75 мин` И `не tp50_hit` И `pnl ≤ 0` И `MFE ≤ 0.40%` | закрытие рынком |
| **Обычный таймаут** | `>150 мин` (SA) | закрытие рынком |
| **Трейлинг** | после TP100 | 0.8 × ATR |

**Почему Smart Timeout существует:** данные показали, что средний MFE «висяков» = 0.22% против 1.31% у TP-сделок. То есть таймаут-сделки мертвы с первых минут, а не «почти доходят». Ранний выход режет средний убыток примерно вдвое и освобождает слот.

## 3. Стратегия SMC — Smart Money Concepts (только BingX, форвард-статистика)

**Идея:** слом структуры (CHoCH) + неотработанный имбаланс (FVG) + торговая сессия (killzone).
**Инструменты:** ~80 альткоинов. **Статистика:** 24 сд, WR 25%, PF 1.70.

Асимметрия результатов: Long PF 2.22 против Short PF 0.66 — редкие крупные победы (avg TP +10.13%) при частых мелких убытках.

### Фильтры (цепочка, каждый отсекает кандидатов)

| Фильтр | Суть | Наблюдение из логов |
|---|---|---|
| `news` | блок перед High-Impact USD-новостями | редко срабатывает |
| `vol` | устойчивый объём окна + минимальный quote-объём + потолок спайка | **режет 63–78 из 80 монет** — главное сито |
| `struct` | наличие структуры (BOS/свинги) | доходят 0–12 |
| `choch` | подтверждённый слом характера | доходят 0–4 |
| `short_blk` | блок шортов в неблагоприятном контексте | 0–3 |
| `vwap` / `rsi` / `adx` | контекстные подтверждения | почти не срабатывают (до них не доходит) |
| `rsi_exhaustion` | отсечение истощённого RSI | — |
| `alt_hi` | alt_score выше порога | — |
| `fvg` | наличие неотработанного FVG | **0 — цепочка рвётся раньше** |

**Вывод по молчанию SMC:** стратегия не сломана. Цепочка рвётся на первом сите (объём) — это низковолатильный рынок, а не пережатые «поздние» фильтры. Отдельный счётчик отсева не нужен: строка `[SMC SCAN]` в логах уже показывает распределение причин.

**Открытый вопрос (v44, не закрыт):** сделка ADA вошла при RSI 66.1, хотя заявленная зона фильтра 55–65. Требуется диагностика, к каким веткам входа применяется RSI-гейт.

## 4. Стратегия RSI (только BingX)

Классический возврат к среднему по альткоинам с мультифильтрами (`mid`, `hook`, `mom`, `sma`, `vwap`, `trend`, `sl`, `sqz`, `pat`). Из логов: объём отсекает 68–80 из 80 монет, до средних фильтров доходят 0–7. Аналогично SMC — молчание объясняется режимом рынка.

## 6. ⚠️ Известная структурная проблема: «мёртвая зона» SA при низком ATR

**Симптом:** с 21 июля ни одного сигнала SA.

**Причина — конфликт двух фильтров.** Поскольку `TP = VWAP`, дистанция до цели равна `|dist_atr| × ATR`, а SL зажат снизу в 1.0%:

```
RR = |dist_atr| × ATR / (price × 1.0%)     [когда ATR < 0.5% цены]
```

При потолке `|dist_atr| ≤ 2.2` максимально достижимый RR равен `2.2 × ATR_% / 1.0%`. Отсюда:

| ATR (% цены) | max RR при 2.2 ATR | Может ли пройти гейт RR ≥ 0.7 |
|---|---|---|
| 0.25% | 0.55 | ❌ нет |
| 0.30% | 0.66 | ❌ нет |
| **0.318%** | **0.70** | граница |
| 0.35% | 0.77 | ✅ да |
| ≥ 0.50% | 1.10 | ✅ да |

**Пока ATR(15m BTC) < 0.318% цены — ни один вход SA не может пройти оба гейта одновременно.** По последней реальной сделке 1 ATR ≈ 0.251% цены → максимум RR 0.55 → мёртвая зона.

**Природа ошибки:** порог `SA_MIN_RR = 0.7` (v44) калиброван на общем WR 51%, который включал бакет 2.2+ (WR 34%), уже заблокированный фильтром v42. После v42 бот торгует только зону 1.5–2.2 ATR с WR 74–80%, где безубыточный RR = 0.25–0.35, а не 0.59. Два фильтра проектировались против разных популяций сделок.

**Дополнительная проблема — слепота:** причины отсева SA логируются на уровне `DEBUG` (строка 4134), в продакшене с уровнем INFO они не видны. Для SMC/RSI есть строки `[SMC SCAN]` / `[RSI SCAN]`, для SA аналога нет.

## 7. Риск-менеджмент и известные ограничения

| Механизм | Значение | Комментарий |
|---|---|---|
| Риск на сделку | 1% депозита | `qty = risk_usdt / sl_dist` |
| Плечо | 5x | |
| Максимум позиций | 2 | |
| Дневной лимит DD | 2.5% | circuit breaker |
| Лимит маржи SA | `MARGIN_PCT_SA = 0.30` (ENV, v45) | потолок notional = `bal × leverage × pct` |
| Лимит маржи прочих | `MARGIN_PCT_ALT = 0.15` (ENV, v45) | 15% блокировал валидные SMC-сетапы с SL > 1.3% |
| Округление лота | `ccxt.ROUND` по шагу биржи | у ccxt дефолт TRUNCATE — терял до 44% позиции |
| Проп-guard риска | пропуск сделки при факт. риске > 1.30× целевого | грубый шаг лота на малом депозите может раздуть риск |

### Ограничения, которые нельзя «починить» кодом

**Неделимый лот BTC.** При депозите ~$78 и риске 1% позиция на Bybit всегда `qty = 0.001` (минимальный лот). TP50 требует закрыть 0.0005 → округляется в ноль → частичная фиксация физически невозможна. Обходной путь (v43): весь объём под БУ. Полноценный TP50 заработает при депозите ≈ $130–150, когда риск 1% даст `qty = 0.002`. Форсировать увеличением риска нельзя: риск 1.68% депозита = 67% дневного лимита DD за одну сделку.

**AI-оракул ненадёжен в числовых утверждениях.** Задокументированы галлюцинации: `"100/100 | RSI > 68"` при фактическом RSI 66.1; та же шаблонная фраза повторялась на разных инструментах. Оракул используется как **advisory**, приоритет всегда у структурных правил (CHoCH и др.). Промпт ужесточён в v44.

**Персистентность БД воркера.** `/data` недоступен для записи → воркер пишет в `./worker_data.db`, который стирается при деплое. Требует монтирования Persistent Disk в Render.

## 8. Методология принятия решений (обязательна)

1. **n ≥ 30 до внедрения фильтра**, n ≥ 15 для «кандидата к наблюдению».
2. **Fisher exact test + Wilson CI.** Пересекающиеся доверительные интервалы = фильтра нет.
3. **Механическое обоснование обязательно.** Цифры без рыночной логики = оверфит (урок PB).
4. **Shadow-валидация** перед выводом стратегии на реальные деньги.
5. **Одно изменение — один деплой** (иначе эффекты смешиваются и причинность теряется).
6. **In-sample ≠ forward.** Зона 1.8–2.2 ATR с WR 80% найдена на тех же данных, на которых измеряется — форвард-валидация обязательна.
7. **Проверять популяцию, на которой калибруется порог** (урок мёртвой зоны §6).
